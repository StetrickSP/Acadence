# Design System Foundation

Read `AGENTS.md`, `architecture.md`, `ui-context.md`, and `code-standards.md` before starting.

This feature establishes the UI foundation for the entire application.

The goal is to install and configure the design system, shared UI primitives, icon library, and utility helpers that every future feature will depend on.

This feature does NOT include application-specific UI.

Do not build dashboards, forms, pages, navigation, analytics screens, or business functionality.

---

## Objective

Set up:

- TailwindCSS
- shadcn/ui
- lucide-react
- reusable class merging utility

Install and configure the core UI primitives used throughout the application.

All components must conform to the dark theme defined in `ui-context.md`.

## Install Dependencies

Install:

```bash
npm install lucide-react
```

Initialize shadcn/ui.

Install the following components:

```bash
npx shadcn@latest add button
npx shadcn@latest add card
npx shadcn@latest add dialog
npx shadcn@latest add input
npx shadcn@latest add tabs
npx shadcn@latest add textarea
npx shadcn@latest add scroll-area
```

---

## Create Utility Helper

Create:

```txt
src/lib/utils.ts
```

Add a reusable `cn()` helper.

Requirements:

- combine Tailwind classes
- support conditional classes
- use `clsx`
- use `tailwind-merge`

Expected exports:

```ts
cn(...)
```

This helper will be the standard class-merging utility across the project.

---

## Theme Requirements

The application is dark-first.

Ensure generated components inherit the existing theme defined in:

```txt
src/globals.css
```

Do NOT introduce:

- default white backgrounds
- default light cards
- bright borders
- light dialog surfaces

All components should automatically consume color tokens defined by the application theme.

## Validation

Verify:

### Imports

The following imports compile successfully:

```ts
import { Button } from "@/components/ui/button"
import { Card } from "@/components/ui/card"
import { Dialog } from "@/components/ui/dialog"
import { Input } from "@/components/ui/input"
import { Tabs } from "@/components/ui/tabs"
import { Textarea } from "@/components/ui/textarea"
import { ScrollArea } from "@/components/ui/scroll-area"
```

---

### Utility

Verify:

```ts
cn("p-4", isActive && "bg-blue-500")
```

returns valid merged classes.

---

### Theme

Verify:

- no light backgrounds appear
- dialogs use dark surfaces
- cards use dark surfaces
- inputs use dark surfaces
- borders match theme tokens

npm run build

## Completion Criteria

3. All required UI primitives exist.
